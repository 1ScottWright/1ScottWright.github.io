Convolution Neural Networks

Keras versus TensorFlow

October 4, 2018

We have been learning about Neural Networks this week in the course. Initially we started out with the Keras package in Python with a straight sequential neural network. We then moved on to convolution additions to the Keras network map with a particular focus on training a computer to recognize images.

Later in the week we began to learn to code Python with just the TensorFlow package without the Keras helper. I think it was beneficial to go down to this level, but I definitely found it much more difficult to use the TensorFlow package directly. Particularly when it comes to using convolutions and pooling.

So, I started thinking (and that usually leads to trouble): if TensorFlow is "more native" and requires more care in setting up, does it run faster than a similar model with the Keras package? I decided to test it out, albeit in a rather unscientific way. I set up a CNN with two layers of convolutions and pools along with an input layer, a hidden layer, and an output layer. I used the Kaggle dataset on digit recognition. The training dataset has 42,000 records with 784 columns. Each column represents a pixel from a 28 x 28 pixel image.

I was surprised to find that the Keras package actually ran about half as fast as the TensorFlow test. I was surprised at the result. If you'd like to check out the code that I used, it is on my GitHub account at the following link: 1ScottWright GitHub
